KernelPS
Previsões para as próximas versões


Nosso objetivo aqui é descrever as funcionalidades nas quais estamos trabalhando para a próxima versão do KernelPS que abrange atualmente os sistemas que foram desenvolvidos apartir dele: (DataWide, Cemisys, Tradeunion, PStudio, PLite e Criatriz).

Não entraremos em detalhes para manter sigilo sobre os novos lançamentos.

Funcionalidades

  • Encapsulamento do núcleo e demais rotinas em classes
  • Objetos/Scripts de kernel destinados aso módulos deverão ser usados de forma única
  • Relatórios estatísticos dos dados armazenados
  • Novos plug-ins com novas funcionalidades
  • Suporte avançado e novo framework para trabalho com AJAX
  • Disponibilização de biblioteca permitindo que o sistema funcione como webservice
  • Pesquisa geral - pesquisa de dado (texto ou num) dentro do banco de dados em qualquer campo ou tabela
  • Unificação modular - rotinas 'dinâmicas' em "_database": data, insupddelmastereditor, etc...

Integrações

  • Pesquisa integrada entre os sistemas do Kernel, formatação de uma API que permita
    pesqueisa gerais, por exemplo, encontrar resultados de um CPF em todso os sistems e BD do Kernel
  • Possibilidade de replicação não nativa do postgree
  • Integração com API Google e possibilidade sub-módulos ligados ao Facebook

Help

  • Manual de operação e configuração de sistemas com o KernelPS
  • Manual de desenvolvimento, regras e frameworks para novos desenvolvedores
